## 1. 项目概述

### 1.1 项目简介

启航电商OMS系统是一个**完整开箱即用的开源电商订单管理系统**，专注核心订单处理业务，支持淘宝天猫、京东、拼多多、抖店、微信小店等主流电商平台。

### 1.2 核心特性

- **多平台支持**：统一管理淘宝、京东、拼多多、抖店、微信小店订单
- **开箱即用**：完整的OMS解决方案，可直接部署使用
- **供应商模式**：支持供应商发货模式
- **电子面单**：支持主流平台电子面单打印
- **API接口**：提供外部API，可对接内部ERP系统
- **权限管理**：完整的RBAC权限控制

| ArtifactId | qihang-oms |
| 开源协议 | AGPL-3.0 |

## 2. 系统架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                         前端展示层 (Vue.js)                          │
│              Vue2 + Element UI + Vuex + Vue Router                  │
├─────────────────────────────────────────────────────────────────────┤
│                         API网关层 (Spring Security)                  │
│                    JWT Token认证 + 权限控制                          │
├─────────────────────────────────────────────────────────────────────┤
│                         控制器层 (Controller)                        │
│            @RestController - 处理HTTP请求，参数校验                   │
├─────────────────────────────────────────────────────────────────────┤
│                         业务服务层 (Service)                         │
│                 @Service - 核心业务逻辑处理                           │
├─────────────────────────────────────────────────────────────────────┤
│                         数据访问层 (Mapper)                          │
│                  MyBatis-Plus - 数据库CRUD操作                       │
├─────────────────────────────────────────────────────────────────────┤
│                         数据存储层                                   │
│                MySQL 8.0 + Redis 7 (缓存/Session)                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 模块依赖关系

```
api (主启动模块)
  ├── core:security (安全认证)
  │     └── core:common (通用工具)
  ├── model (实体模型)
  ├── mapper (数据访问)
  │     └── model
  └── service (业务服务)
        ├── model
        └── mapper
```

### 2.3 分层架构说明

| 层级 | 模块 | 职责 |
|------|------|------|
| **API层** | api | 接收HTTP请求，参数校验，调用服务层 |
| **服务层** | service | 业务逻辑处理，事务管理 |
| **数据层** | mapper | 数据库操作，SQL执行 |
| **模型层** | model | 实体类、VO、DTO、查询对象 |
| **安全层** | core:security | 认证授权、Token管理 |
| **通用层** | core:common | 工具类、配置、异常处理 |

## 4. 功能流程

### 4.1 核心业务流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  电商平台    │───▶│  订单拉取    │───▶│  订单处理    │───▶│  发货管理    │
│ (淘宝/京东等) │    │  (API同步)   │    │  (审核/分配) │    │  (电子面单)  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                │
                                                                ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  数据统计    │◀───│  账单管理    │◀───│  物流跟踪    │◀───│  售后处理    │
│  (报表分析)  │    │  (对账结算)  │    │  (状态更新)  │    │  (退换货)    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 4.2 订单处理流程

```
1. 店铺授权 → 2. 订单拉取 → 3. 订单审核 → 4. 分配发货 → 5. 打印面单 → 6. 确认发货 → 7. 物流跟踪
```

### 4.3 商品管理流程

```
1. 商品录入 → 2. SKU管理 → 3. 库存设置 → 4. 店铺商品关联 → 5. 价格同步
```

### 4.4 售后处理流程

```
1. 售后申请 → 2. 售后审核 → 3. 退货/换货/退款 → 4. 库存回滚 → 5. 财务处理
```

## 6. 项目快速启动

### 6.1 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| **JDK** | 17+ | Java开发环境 |
| **Maven** | 3.8+ | 项目构建工具 |
| **Node.js** | v16.x | 前端运行环境 |
| **MySQL** | 8.0 | 数据库 |
| **Redis** | 7 | 缓存服务 |

### 6.2 后端启动步骤

#### 步骤1：创建数据库

```sql
-- 创建数据库
CREATE DATABASE `qihang-oms` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 导入SQL脚本
-- 使用Navicat或其他工具导入 docs/qihang-oms.sql
```

#### 步骤2：启动Redis

```bash
# Windows
redis-server.exe

# Linux/Mac
redis-server
```

#### 步骤3：修改配置文件

编辑 `api/src/main/resources/application.yaml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/qihang-oms?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
    username: root        # 修改为你的数据库用户名
    password: your_password  # 修改为你的数据库密码
  data:
    redis:
      host: 127.0.0.1     # Redis地址
      port: 6379          # Redis端口
      password:           # Redis密码（如有）
```

#### 步骤4：Maven打包

```bash
# 进入项目根目录
cd D:\开源项目\qihang-oms

# Maven打包（跳过测试）
mvn clean package -DskipTests
```

#### 步骤5：启动后端

```bash
# 方式1：命令行启动
java -jar api/target/api-2.2.0.jar

# 方式2：IDE中启动
# 运行 api/src/main/java/cn/qihangerp/api/ApiApplication.java
```

### 6.3 前端启动步骤

#### 步骤1：安装依赖

```bash
# 进入前端目录
cd vue

# 安装依赖（使用国内镜像加速）
npm install --registry=https://registry.npmmirror.com
```

#### 步骤2：开发模式启动

```bash
npm run dev
```

#### 步骤3：生产打包

```bash
npm run build:prod
```

### 6.4 访问系统

| 项目 | 地址 |
|------|------|
| **前端访问** | http://localhost:88 |
| **后端API** | http://localhost:8086 |
| **默认账号** | admin |

### 6.5 生产环境部署（Nginx配置）

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html index.htm;
        try_files $uri $uri/ /index.html;
    }

    # 后端API转发
    location /prod-api/ {
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header REMOTE-HOST $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass http://localhost:8086/;
    }
}
```

## 8. 开发指南

### 8.1 新增API接口

1. 在 `model` 模块创建实体类（entity）
2. 在 `mapper` 模块创建Mapper接口
3. 在 `service` 模块创建Service接口和实现
4. 在 `api` 模块创建Controller

### 8.2 新增前端页面

1. 在 `vue/src/api/` 创建API接口

## 9. 常见问题

### Q1：启动报数据库连接失败？

**A**：检查 `application.yaml` 中的数据库配置是否正确，确保MySQL服务已启动。

### Q2：启动报Redis连接失败？

**A**：确保Redis服务已启动，检查Redis配置是否正确。

### Q3：前端启动报npm install失败？

**A**：尝试使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

### Q4：登录后页面空白？

**A**：检查后端API是否正常启动，查看浏览器控制台网络请求是否成功。

### Q5：如何新增一个电商平台？

**A**：
1. 参考 `api/tao` 目录结构创建新平台目录
2. 创建对应的实体类、Mapper、Service、Controller
3. 前端创建对应页面和路由

### Q6：如何修改端口？

**A**：
- 后端：修改 `application.yaml` 中的 `server.port`
- 前端：修改 `vue.config.js` 中的 `devServer.port`
